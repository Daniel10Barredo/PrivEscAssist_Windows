Grupos del usuario

EL segundo paso cosiste en comprobar los grupos a los que pertenece el usuario. Esta comprobación es de suma importancia, ya que algunos de estos grupos podrían representar posibles vías directas para llevar a cabo la escalada de privilegios.

Comprobación

Utiliza el siguiente comando para comprobar los grupos a los que pertenece el usuario.

whoami /groups

Los grupos con privilegios que puede aprovechar un atacante para una escalada de privilegios son los siguientes:

Administrators, Domain Admins, Enterprise Admins Backup Operators DnsAdmins Exchange Windows Permissions Hyper-V Administrators Organization Management Print Operators Account Operators Server Operators
