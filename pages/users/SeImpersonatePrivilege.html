Impersonate Privilege

◍ Explicación

Esta directiva que suele estar asignada a cuentas de servicio, permite suplantar a un usuario u otra cuenta específica y actuar en su nombre, más concretamente que cuentas de usuario puede llamar a CreateProcessAsUser() de la API de Windows para que un servicio pueda iniciar a otro.

◍ Ataque

Para la explotación de este tipo de privilegios, son muy conocidos en la comunidad los ataques tipo Juicy Potato y sus variantes. Estos ataques se basan en la combinación de una suplantación de identidad NBNS y al mismo tiempo una retransmisión NTLM de forma interna para crear un servicio con las credenciales que ejecute un comando definido por el usuario.

Ha habido algunos cambios por parte de Microsoft tratando de bloquear este tipo de ataques, aunque por compatibilidad todavía hay variantes que funcionan a día de hoy. La versión Juicy Potato es funcional hasta Windows 10 build 1809 y Windows Server 2019 y para versiones superiores habría que utilizar otra variante como Rouge Potato .

EJEMPLO

Comprobamos que tenemos el permiso seImpersonatePrivilege disponible.

Como ya hemos visto habría que utilizar un ataque de tipo Potato pero necesitamos comprobar previamente el sistema objetivo con systeminfo.

Así que tenemos un Windows 7 SP1 y para esto podemos utilizar simplemente juicy-potato con el que ejecutaremos una shell reversa previamente creada.

Y rápidamente obtenemos la conexión con la shell reversa ejecutada como SYSTEM.
